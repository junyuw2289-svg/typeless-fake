## 1. 产品概览

**Typeless** 是一款 AI 驱动的语音听写（Voice Dictation）应用，核心理念是 **"Speak, don't type"（说话，不打字）**。它将自然语音实时转化为经过润色的书面文字，声称比传统打字快 4-6 倍（220 WPM vs 45 WPM）。

## 3. 核心功能详解

### 3.1 智能语音听写（Core Dictation）

Typeless 不是简单的语音转文字，而是通过 LLM（大语言模型）对语音输入进行智能处理：

- **自动去除填充词**：自动删除 "um"、"uh"、"you know" 等口头禅
- **自动去除重复**：检测并移除不必要的重复词语
- **自我纠正识别**：当用户说话中途改口时，只保留最终意图的内容
- **自动格式化**：将口述的列表、步骤、要点组织为干净的结构化文本
- **自动标点**：智能添加标点符号、大写字母
- **特殊格式识别**：自动格式化电话号码、电子邮件等结构化数据

### 3.2 语音编辑（Speak to Edit）

选中任意文本后，用语音说出想要的修改：
- 缩短/加长内容
- 改变语气
- 修改格式
- 修复错别字
- 创建回复

### 3.3 语音提问（Speak to Ask）

对只读内容（如网页、文档）用语音进行操作：
- 总结
- 解释
- 翻译
- 快速回答

### 3.4 上下文感知（Context-Aware Tone）

根据当前使用的应用自动调整输出的语气和风格：
- **Gmail/Outlook**：正式、专业的邮件语气
- **Slack/WhatsApp**：轻松、对话式语气
- **VS Code/Cursor**：技术性语言，识别代码语法和变量名
- **文档工具**：结构化写作风格

### 3.5 个性化学习（Personalization）

- 随时间学习用户的语气、措辞和写作习惯
- 提供"个性化进度"追踪，显示与用户沟通风格的对齐程度
- 据用户反馈，使用 19 天后明显学会了用户的语音模式和词汇
- 使用越多，适应效果越好

### 3.6 多语言支持

- 支持 **100+ 语言**
- **自动语言检测**：无需手动切换
- **语言混合**：可在同一段话中无缝切换语言
- **即时翻译**：说话的同时翻译为目标语言，用词像本地人一样自然

### 3.7 个人词典（Custom Dictionary）

- 添加专业术语、品牌名称、人名
- 确保特定词汇的一致准确转录

### 3.8 语音指令

支持基本语音命令：
- "New line"（换行）
- "New paragraph"（新段落）
- "Comma"（逗号）
- "Period"（句号）
- "Question mark"（问号）

## 4. 交互模式与工作流程

### 4.1 桌面端（macOS/Windows）

**激活方式：Push-to-Talk（按住说话）**

1. 将光标放在任意文本输入框中
2. **按住** 快捷键（默认为 `Fn` 键）开始说话
3. **松开** 快捷键，AI 处理并输出润色后的文本
4. 文本直接插入到光标位置

**快捷键配置**：
- 默认：`Fn`（Function 键，Mac 键盘左下角）
- 推荐：`Right Command`（Mac）或 `Right Alt`（Windows）
- 可自定义为其他快捷键

**桌面端特点**：
- 系统级应用，在后台静默运行
- 无需打开应用界面即可使用
- 需要 **Accessibility（辅助功能）** 和 **Microphone（麦克风）** 权限
- 适用于所有应用和网站，无需复制粘贴

### 4.2 移动端（iOS/Android）

**作为键盘替代**：
- 安装后替换系统键盘
- 文本输入框激活时显示 Typeless 键盘
- 界面中心是一个 **大型麦克风图标**
- 其他控件：`@` 符号、空格键、删除键、回车键

**使用流程**：
1. 点击文本输入框，显示 Typeless 键盘
2. 点击麦克风图标开始录制
3. 自然说话（最长可持续 6 分钟）
4. 再次点击麦克风图标停止，约 1-2 秒处理
5. 润色后的文本插入输入框

**局限性**：
- Android 版 **不包含虚拟键盘**，需要在 Typeless 和另一个键盘之间切换来手动输入

## 5. UI/UX 设计分析

### 5.1 整体设计风格

- **极简主义**：干净、无干扰的界面
- **字体**：使用 Inter 字体家族
- **配色**：
  - 主色：深炭色 `rgba(29, 26, 26, 1)` / 接近黑色
  - 背景强调色：浅灰 `rgba(245, 245, 245, 1)`
  - 整体偏向暗色调 + 浅色辅助
- **字重层级**：400-700 不等
- **圆角**：大圆角设计（24-32px border-radius）

### 5.2 交互元素

- **药丸形按钮**：带 hover 状态
- **图标控件**：20-30px 大小，SVG 基础的图标系统
- **语言选择器**：带动画的下拉菜单
- **浮动操作按钮**：快速访问入口

### 5.3 录制状态指示

- **波形动画（Wave Animation）**：录制时显示音频波形动画
- **蓝色活动条（Blue Activity Bar）**：麦克风设置时可见，用于确认麦克风正在工作
- 视觉反馈清晰，解决了"不确定麦克风是否在监听"的痛点

### 5.4 布局特点

- **粘性头部导航（Sticky Header）**
- **滚动吸附（Scroll-snap）**：平滑滚动体验
- **响应式布局**：移动端单列 → 桌面端多列
- **间距系统**：8px 的倍数
- **大量留白**：强调清晰度

### 5.5 技术实现线索

- 使用 Google Analytics / Google Tag Manager
- Material-UI 组件系统
- CSS-in-JS 样式方案
- 复杂的媒体查询断点
- 动画序列用于视觉反馈

## 6. 隐私与安全

Typeless 强调 **"Privacy First"（隐私优先）** 架构：

- **零数据留存**：语音和听写数据不存储在云端
- **不训练模型**：用户数据永远不会用于 AI 模型训练
- **本地存储**：听写历史仅存储在用户设备上
- **可控历史**：用户可配置历史记录保留期限

这是相对于竞品 Wispr Flow（使用 OpenAI 和 Meta 的云服务）的一个关键差异化优势。

## 11. 关键设计理念总结

1. **智能自动化优先于手动干预**：AI 负责润色，用户只需自然说话
2. **意图理解而非字面转录**：理解用户想说什么，而不是逐字记录
3. **上下文感知**：根据使用场景自动调整输出风格
4. **隐私优先**：所有处理在本地完成，零云端数据留存
5. **渐进式个性化**：持续学习用户的写作风格和偏好
6. **最小化 UI**：界面几乎"隐形"，让用户专注于说话内容
7. **系统级集成**：作为操作系统级别的输入方式，而非独立应用

## 12. 对我们仿制项目的关键启示

### 必须实现的核心体验
1. **Push-to-Talk 快捷键**：按住说话，松开处理（默认 Fn 键，可自定义）
2. **实时语音转文本**：使用 LLM 润色输出
3. **录制状态的视觉反馈**：波形动画 + 活动指示器
4. **系统级文本插入**：直接将结果插入到任意应用的光标位置
5. **去填充词和自我纠正**：输出比说话内容更清晰

### UI 设计要点
- 极简风格，深色主题为主
- 大圆角（24-32px）
- Inter 字体
- 波形动画作为录制反馈
- 药丸形按钮
- 后台运行，最小化 UI 干扰

### 技术关键点
- 需要 Accessibility（辅助功能）权限实现系统级文本插入
- 需要 Microphone（麦克风）权限
- Push-to-Talk 需要全局快捷键监听
- 本地处理优先的架构
